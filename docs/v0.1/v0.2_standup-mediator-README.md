# 🎙️ Skarya Pulse — `features/standup-mediator`

> An LLM-powered daily standup facilitator for [skarya.ai](https://skarya.ai).  
> Kobi runs the standup conversation, understands free-form responses, and automatically updates your boards — no manual data entry, no custom NLP pipeline.

---

## 🧠 Design Philosophy

This is an **LLM-native** feature. There is no custom NLP classifier, no intent schema to maintain, no ML model to train.

```
Chat UI  →  LLM API (with skarya context injected)  →  Structured JSON output  →  skarya.ai APIs  →  Done.
```

The LLM is the brain. skarya.ai's existing internal APIs are the hands. Our job is wiring them together cleanly.

## 🔄 How It Works

1. **Session starts** → fetch user's tasks from skarya.ai internal API
2. **Inject context** → tasks + user info injected into LLM system prompt
3. **LLM runs conversation** → asks Q1, Q2, Q3, handles follow-ups and ambiguity naturally
4. **Structured output** → LLM returns a JSON object: task updates, new tasks, blockers, summary
5. **User confirms** → confirmation card shown, user approves or edits
6. **Write-back** → confirmed changes written to skarya.ai via internal APIs
